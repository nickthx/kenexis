    **File 1: Update `src/app/(marketing)/layout.tsx`**

    Replace the current placeholder layout with the full marketing layout shell:

    ```typescript
    import { ViewTransitions } from "next-view-transitions";
    import { SiteHeader } from "@/components/layout/site-header";
    import { SiteFooter } from "@/components/layout/site-footer";
    import { SiteBreadcrumbs } from "@/components/layout/breadcrumbs";

    export default function MarketingLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      return (
        <ViewTransitions>
          <SiteHeader />
          <SiteBreadcrumbs />
          <main id="main-content" className="min-h-screen pt-16">
            {children}
          </main>
          <SiteFooter />
        </ViewTransitions>
      );
    }
    ```

    Key details:
    - `pt-16` on main offsets content below the fixed header (h-16 = 4rem = 64px). This prevents the pitfall where content renders behind the fixed header.
    - `min-h-screen` ensures the footer stays at the bottom even on short pages.
    - The layout is a SERVER component (no "use client"). ViewTransitions from next-view-transitions is a server component wrapper.
    - SiteHeader and SiteBreadcrumbs are client components (they use hooks), but they can be rendered inside a server component layout.

    **File 2: Append to `src/app/globals.css`**

    Add view transition CSS animations at the END of the file (after the existing `.dark` block and `@layer base` block):

    ```css
    /* Page transitions via next-view-transitions */
    ::view-transition-old(root) {
      animation: fade-out 150ms ease-out;
    }

    ::view-transition-new(root) {
      animation: fade-in 150ms ease-in;
    }

    @keyframes fade-out {
      from { opacity: 1; }
      to { opacity: 0; }
    }

    @keyframes fade-in {
      from { opacity: 0; }
      to { opacity: 1; }
    }

    /* Respect user motion preferences */
    @media (prefers-reduced-motion: reduce) {
      ::view-transition-old(root),
      ::view-transition-new(root) {
        animation: none;
      }
    }
    ```

    IMPORTANT: Per the next-view-transitions library docs, the `<Link>` component from `next-view-transitions` IS needed to trigger transitions. The wrapper alone is not sufficient. So update the mega-menu, mobile-nav, breadcrumbs, site-header, and site-footer components to use `import { Link } from "next-view-transitions"` instead of `import Link from "next/link"` for all internal navigation links. External links (target="_blank") should continue using regular `<a>` tags.

    Update these files to swap the Link import:

    - `src/components/layout/site-footer.tsx`: Same swap for internal sitemap links (keep `<a>` for external links)
