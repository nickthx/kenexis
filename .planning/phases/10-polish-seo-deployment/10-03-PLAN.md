  <action>
1. Check Vercel CLI auth status: `vercel whoami`
   - If authenticated, proceed
   - If not authenticated, this will become a checkpoint for the user to run `vercel login`

2. Run `vercel --yes` to link the project and do a preview deployment. The `--yes` flag auto-confirms project setup prompts.

3. Verify the preview deployment loads by checking the URL output.

4. Run `vercel --prod` to promote to production deployment.

5. Record the production URL (format: `project-name.vercel.app`).

6. Verify the production URL loads the homepage by checking the response.

Note: The `metadataBase` in layout.tsx uses `process.env.VERCEL_PROJECT_PRODUCTION_URL` which Vercel auto-sets during build. No manual URL update needed -- the env var is automatically available in Vercel's build environment. This avoids the chicken-and-egg problem described in research.
